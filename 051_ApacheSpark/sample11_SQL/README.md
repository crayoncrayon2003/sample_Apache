# sample11_SQL — Spark SQL / DataFrame

**Spark SQL** is one of Spark's four main parts.
It handles structured data as a DataFrame. This folder holds two independent patterns
that show the same idea in two styles. The two series are a matching pair: each one has a
`generate` step, an `input` folder, a `transform` step, and an `output` folder.

## Pattern 1 (0x) — DataFrame API
```bash
python3.12 01_generateDummyTaxiData.py
python3.12 02_dataframe_api.py
```

## Pattern 2 (1x) — SQL string

Runs a SQL string on a temp view (`createOrReplaceTempView` + `spark.sql()`).
Also shows how to save and restore a schema.

```bash
python3.12 11_generateDummyDataSet.py
python3.12 12_spark_sql.py
```

## Folder naming

- `00_input` / `00_output` — input and output for pattern 1.
- `10_input` / `10_output` — input and output for pattern 2.
- All four folders are generated by the scripts, so they are not tracked in git.
